RESEARCH ASSISTANT

UNC Poplov Lab

HPC systems + MLOps for large-scale simulation workflows

Sep 2024 – Present PI: Prof. Nikolay Poplov UNC Chapel Hill Platform: UNC Longleaf HPC

The Problem

Large-scale molecular dynamics simulations on HPC clusters involve painful manual workflows: preparing systems, submitting Slurm jobs, monitoring convergence, handling failures, and post-processing results. Researchers spend significant time on infrastructure rather than science. Additionally, many simulations run longer than necessary because there is no automated way to detect convergence.

HPC Automation Platform

Designed and deployed a Slurm-based orchestration framework managing the full simulation lifecycle:

System Preparation & Submission

  • Automated topology generation, solvation, and equilibration workflows
  • Template-based Slurm script generation with resource optimization
  • Batch submission with dependency chains (equilibration → production → analysis)

Monitoring & Fault Tolerance

  • Checkpoint/resume system for long-running simulations
  • Automatic detection and restart of failed jobs
  • Log aggregation and alerting for anomalous behavior
  • Reduced manual configuration errors significantly

Convergence Prediction

Trained convergence predictors on energy and RMSD streams:

  • XGBoost: Fast prediction from windowed statistical features (rolling mean, variance, autocorrelation)
  • LSTM: Sequence-based prediction for detecting convergence trends
  • Result: ~35% reduction in wasted compute through intelligent early stopping

Reproducible Environments

Containerized workflows (Docker + Conda) with standardized configs, logging, and fault-tolerant restarts. Reduced onboarding time from ~4 hours to less than 20 minutes. Eliminated environment drift across machines.

CI/CD for Science

Integrated GitHub Actions for automated environment validation and unit tests. Enforced version control and regression checks for simulation and analysis codepaths.
